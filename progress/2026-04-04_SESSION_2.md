## Gotchas & Lessons Learned (Session 2)

1. **OpenCode is x86 only (Docker image)**: Cannot run on DGX Spark (ARM64). Runs on Minisforum sending inference to DGX Spark GPU.

2. **OpenCode needs explicit `serve` command**: The Docker image entrypoint is `opencode` but needs `serve --hostname 0.0.0.0 --port 3002` as args. Without it, the container starts but doesn't serve on any port.

3. **APISIX admin key changes per container**: After recreating the container, always check `/tmp/apisix-config.yaml` for the actual key.

4. **APISIX Prometheus metrics need `plugin_attr` in config.yaml**: The global rule alone isn't enough. The config file needs `prometheus` under `plugin_attr` with `export_addr` settings.

5. **`@laceletho/plugin-openclaw` does not exist**: The architecture guide hallucinated this npm package. OpenClaw has native multi-agent support via `openclaw agents add`, `models aliases`, `models fallbacks`.

6. **OpenClaw real commands**: `agents add` (not `spawn`), `models aliases add`, `models fallbacks add`, `models set`. Always check `--help` before running guide commands.

7. **SCP is the reliable way to edit configs**: SSH terminal pastes break Python scripts and JSON. Write files on Mac, SCP them over.
