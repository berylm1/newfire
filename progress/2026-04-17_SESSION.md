## Executive Summary

Major architectural milestone: built and deployed the backend service that coordinates all platform systems. The platform went from hardcoded demo accounts and localStorage to a real database-backed system with JWT auth, onboarding pipeline, and OpenClaw orchestration. Every chat message now routes through OpenClaw with three layers of fallback.

**Key Accomplishments:**

- Built Node.js backend service coordinating PostgreSQL + OpenClaw + APISIX
- Real user signup/login with hashed passwords and JWT tokens in PostgreSQL
- Onboarding pipeline: AI interview generates agents, creates company in database, provisions APISIX consumer
- Enabled OpenClaw HTTP chat completions endpoint (was disabled by default)
- All chat routes through OpenClaw with fallback: OpenClaw -> direct Ollama -> OpenRouter free tier
- Rewrote entire React frontend to use backend API instead of localStorage/hardcoded data
- Transcribed Ms. Sherifah's 8-minute voice recording via Whisper, rewrote all agent prompts for financial services use case
- Removed redundant Ollama CPU models from Minisforum (18GB freed, disk 81% to 61%)
- Permanent log rotation configured (500MB cap, 7-day retention)
- Added markdown rendering to chat bubbles
- MiniMax M2.7 cloud model pulled on DGX Spark
- Removed basic auth gate (app now has its own auth system)
- Identified and fixed OpenRouter credits exhaustion (fallback to local models)

## Architecture (Current)

```
User Browser
    |
Cloudflare Tunnel (TLS, DDoS protection)
    |
nginx (:4000, Docker container)
    |
    +-- /backend/* -> Backend Service (:3200)
    |                    |
    |                    +-- PostgreSQL (:5433) for users, companies, agents, conversations
    |                    |
    |                    +-- OpenClaw (:18789) for chat orchestration + fallback
    |                    |       |
    |                    |       +-- DGX Spark Ollama (:11434) for GPU inference
    |                    |       +-- OpenRouter (cloud fallback)
    |                    |
    |                    +-- APISIX (:9180) for consumer/key management
    |
    +-- /api/* -> APISIX (:9080) for developer API access
    +-- /admin/* -> APISIX admin (key injected server-side)
```

## Backend API Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| /backend/health | GET | Health check |
| /backend/auth/signup | POST | Create user account |
| /backend/auth/login | POST | Authenticate, return JWT |
| /backend/auth/me | GET | Get current user |
| /backend/company | GET | Get user company and agents |
| /backend/company/create | POST | Create company with agents |
| /backend/agents | GET | List user agents |
| /backend/chat/:agentId | POST | Chat (routes through OpenClaw) |
| /backend/chat/:agentId/history | GET | Get saved conversation |
| /backend/onboarding/chat | POST | Onboarding interview |
| /backend/onboarding/activate | POST | Parse agents, create company |

## Chat Routing (3 layers of fallback)

```
1. OpenClaw (:18789/v1/chat/completions)
   - Full orchestration, model routing, agent system
   |
   v (if fails)
2. Direct Ollama (DGX Spark :11434)
   - Bypass orchestration, direct model inference
   |
   v (if fails)
3. OpenRouter (Nemotron Nano 30B, free tier)
   - Cloud fallback, always available
```
