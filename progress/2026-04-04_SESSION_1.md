### Phase 0: Fix DGX Spark Access

**Problem**: Could not SSH into the DGX Spark. Password authentication kept failing despite correct credentials.

**Root Cause**: Unknown — password auth was enabled in SSH config but login kept being denied.

**Solution**: Bypassed password auth entirely with SSH key-based authentication.

```bash
# On Mac — generate SSH key (didn't have one)
ssh-keygen -t ed25519
# Press Enter for all defaults

# Copy the public key
cat ~/.ssh/id_ed25519.pub
# Copied the output

# On DGX Spark (keyboard/monitor) — add the key
mkdir -p ~/.ssh && chmod 700 ~/.ssh
echo "PASTE_PUBLIC_KEY_HERE" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

# From Mac — SSH now works without password
ssh newwave-dgx@100.88.112.5
```

**Tailscale IPs established**:
- Minisforum: `100.79.80.119`
- DGX Spark: `100.88.112.5`
- Username on Spark: `newwave-dgx`

### Phase 8: Install Ollama on DGX Spark + Pull Models

```bash
curl -fsSL https://ollama.com/install.sh | sh
sudo systemctl enable --now ollama

# Configure for network access
sudo mkdir -p /etc/systemd/system/ollama.service.d
sudo tee /etc/systemd/system/ollama.service.d/override.conf << 'EOF'
[Service]
Environment="OLLAMA_HOST=0.0.0.0"
Environment="OLLAMA_NUM_PARALLEL=4"
Environment="OLLAMA_MAX_LOADED_MODELS=3"
EOF
sudo systemctl daemon-reload
sudo systemctl restart ollama

# Pull models
ollama pull deepseek-r1:70b   # 42 GB — too large for full context
ollama pull deepseek-r1:32b   # 19 GB
ollama pull glm4:9b           # 5.5 GB
```

**Critical issue**: `deepseek-r1:32b` crashed with "out of memory" because Ollama allocated a 77 GB KV cache (524K context window). Fixed by creating a context-limited variant:

```bash
echo 'FROM deepseek-r1:32b
PARAMETER num_ctx 8192' > /tmp/Modelfile
ollama create deepseek-r1:32b-8k -f /tmp/Modelfile
# Now works — uses ~20 GB instead of 77+ GB
```

## Gotchas & Lessons Learned

1. **SSH terminal mangles long commands**: Multi-line pastes over SSH get broken. Workaround: write files on Mac and SCP them, or use single-line commands.

2. **NodeSource apt conflict**: Two repo files with different `Signed-By` paths. Fix: `sudo rm /etc/apt/sources.list.d/nodesource.sources` (keep the `.list` file).

3. **OpenClaw `openclaw` user has no sudo password**: Install system packages as `newwaveclaw` first, then switch to `openclaw` for OpenClaw commands.

4. **Ollama KV cache explosion**: Default context window (524K) causes OOM on large models. Fix: create context-limited model variants with `PARAMETER num_ctx 8192` in a Modelfile.

5. **OpenClaw config validation is strict**: Must provide all required fields together (e.g., `models` array required when creating a provider). Cannot set fields incrementally via `openclaw config set`.

6. **OpenClaw api field**: Must be one of the enum values (`openai-responses`, `ollama`, etc.), not just `openai`.

7. **NemoClaw port 8080 is gRPC, not REST**: Cannot curl it directly. Cross-machine communication goes through Ollama port 11434.

8. **APISIX quickstart admin key**: Randomly generated per container — check `config.yaml` for the actual key. Also must change `allow_admin` to access from Docker host.

9. **DGX Spark username**: `newwave-dgx` (not `newwave-gdx`) — the hyphen placement matters.
